üìò Introduction to RPC

Remote Procedure Call (RPC) is a protocol that allows a program to execute a procedure on another address space (commonly on another physical machine) as if it were a local procedure call, abstracting the network communication details from the developer.

üîç RPC Architecture

  • Client: Initiates the RPC by requesting a remote procedure.
  • Client Stub: Marshals the procedure parameters and sends them to the server.
  • RPC Runtime: Handles the transmission of data between client and server.
  • Server Stub: Unmarshals the parameters and invokes the procedure on the server.
  • Server: Executes the procedure and returns the result to the client.

Advantages:

  • Abstracts network communication, simplifying distributed application development.
  • Supports multiple programming languages, facilitating integration across diverse systems.
  • High performance with support for streaming, suitable for real-time applications.

Disadvantages:

  • Network failures can disrupt communication, requiring robust error handling mechanisms.
  • Security concerns, necessitating encryption and authentication strategies.

  • Apache Thrift: A cross-language RPC framework that supports multiple protocols and data formats.
  • JSON-RPC: A lightweight RPC protocol that uses JSON for encoding and HTTP for transport.
